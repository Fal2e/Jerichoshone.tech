1. Physical Layer


The first layer in the OSI model, the physical medium to move data between devices.
Such as modems, hubs, cat6 cables.



Key responsibilities of the physical layer are encoding and signaling,
physical data transmission, hardware specifications, topology & diagram. The data type is bits.


2. Datalink Layer


The Datalink layer contains the Logical Link Control(LLC), Media Access Control (MAC), Data Framing, Addressing, Error Detection. These protocols are generally responsible to simply ensure and confirm that the bits and bytes that are received are identical to the bits and bytes being transferred. It is basically a set of specifications that are used for implementation of data link layer just above the physical layer. Key componants to the datalink layer are Switches, MAC, Ethernet, NIC, Tunneling-enapsulation.



Attack vecters to focus on when securing the Datalink Layer. Port Security, Disabling Dynamic Trunking Protocol (DTP), Dynamic Host Configuration Protocol (DHCP) Snooping, Private Virtual Area Network (VLANS), Spanning Tree Protocol (STP).


3. Network Layer


The Network is comprised of logical address, routing, path determination,
datagram, encapsulation, error handling & Diagnostics. The datatype is packets/datagrams.



A secured network is the first line of defence for web application's. Threats such as network sniffing, denial of server (DOS) attacks, spoofing.. The list is ever growing.


Best Practice is to config your router, firewall, and switches correctly.

Router Security


  • Router operating system up to date
  • Unused ports are blocked or 'down'
  • Unused interfaces and services are disabled
  • Logging and auditing of unusual activity
  • Packet filtering
  • IDS & IPS enabled

Firewall Security


  • Software is updated daily
  • Firewall placement
  • Logging & auditing for any unusual activity
  • Packet filtering

Switch Security


  • Switch traffic is encrypted
  • OS / Firmware is up to date
  • Unused interfaces and services are disabled

4. Transport Layer


In computer networking, the transport layer is a conceptual division of methods in the layered architecture of protocols in the network stack in the Internet Protocol Suite and the Open Systems Interconnection (OSI). The best-known transport protocol of TCP/IP is the Transmission Control Protocol (TCP) and User Datagram Protocol (UDP).


Transport Layer Services

  • Connection-oriented communication
  • Reliability
  • Same order delivery
  • Flow control
  • Port multiplexing
  • Congestion avoidance


Transport Security Layer


The TLS protocol is the successor of the Secure Sockets Layer (SSL) protocol. Netscape originally developed SSL in 1994 to secure its browser, Netscape Navigator. The last version of SSL was SSL 3.0. The first version of TLS, released in 1999, is based on SSL 3.0.

5. Session Layer


Layer five provides inter-process communication between two systems, controling the logical connections. The session layer provides the mechanism for opening, closing and managing a session between end-user application processes.


An example of a session-layer protocol is the OSI protocol suite session-layer protocol, also known as X.225 or ISO 8327. In case of a connection loss this protocol may try to recover the connection. If a connection is not used for a long period, the session-layer protocol may close it and re-open it.

6. Presentation Layer


De/Encryption, Encoding, String representation


The presentation layer (data presentation layer, data provision level) sets the system-dependent representation of the data (for example, ASCII, EBCDIC) into an independent form, enabling the syntactically correct data exchange between different systems. Also, functions such as data compression and encryption are guaranteed that data to be sent by the application layer of a system that can be read by the application layer of another system to the layer 6. The presentation layer. If necessary, the presentation layer acts as a translator between different data formats, by making an understandable for both systems data format, the ASN.1 (Abstract Syntax Notation One) used.

7. Application Layer


The definition of the application layer is narrower in scope. The OSI model defines the application layer as the user interface responsible for displaying received information to the user.
